## 1. The Asymptotic Triad: O, Ω, and Θ

Imagine algorithm complexity is a function f(n). Notations serve to "envelope" this function.

* **Big-O Notation (O):** The **upper bound**. It guarantees the algorithm will *never* be worse than this. If code is O(n²), it may take n or n² steps, but never n³. This is what we use for **Worst Case**.
* **Omega Notation (Ω):** The **lower bound**. It guarantees the algorithm is *at least* as costly as this. This is what we use for **Best Case**.
* **Theta Notation (Θ):** The **tight bound**. We say an algorithm is Θ(g(n)) if it is simultaneously O(g(n)) and Ω(g(n)).

**When are they equal?**
When algorithm behavior **depends only on input size n, not on data arrangement**.
* *Example:* Code that sums all array elements. Whether numbers are large, small, or sorted doesn't matter; you **always** traverse n elements. Thus, this is Θ(n).

**When are they different?**
When "luck" influences execution.
* *Example:* **Linear Search**.
    * Element in first position: Ω(1) (Best Case).
    * Element not in array: O(n) (Worst Case).
    * Since 1 ≠ n, we can't say Linear Search is Θ(n) generically.

## 2. Execution Properties: In-place and Stability

These terms define algorithm "quality" and "behavior," especially in sorting.

* **In-place Algorithm:**
    * Uses no extra memory proportional to input size.
    * **Space Complexity:** O(1) auxiliary memory.
    * *Analogy:* Organizing bookshelf by moving books within the shelf, needing no extra table.

* **Stable Algorithm:**
    * Preserves relative order of equal elements.
    * *Analogy:* If two items tie in sorting criteria, they keep their original order.

### 🍎 Practical Example: The Fruits Case

Initial list (sorted by name):
1. (Apple, $10)
2. (Banana, $5)
3. (Pear, $10)

**Stable Algorithm** (sort by price):
1. (Banana, $5)
2. (Apple, $10)
3. (Pear, $10)

Apple stays before Pear because they tied in price.

**Unstable Algorithm** (sort by price):
1. (Banana, $5)
2. (Pear, $10)
3. (Apple, $10)

Order was disrupted in the tie.

## 3. Best Case vs. Worst Case

* **Best Case:** Input configuration making code reach the end fastest.
* **Worst Case:** Input configuration forcing maximum instructions.

Example: Insertion Sort on already-sorted array = O(n) (best); reverse-sorted = O(n²) (worst).

---

## 4. Graphs: Adjacency List

**Forget the matrix.** Adjacency List stores only existing connections.
* **Structure:** Array of size V (vertices), each pointing to its neighbors' list.
* **Space Cost:** O(V + E).
* **Why it matters:** Real graphs are sparse; matrices waste space with zeros.

---

## 5. Traversal Algorithms

* **BFS (Breadth-First):** Explores level by level. Uses **Queue**. O(V + E).
* **DFS (Depth-First):** Explores deep before backtracking. Uses **Stack** or recursion. O(V + E).
